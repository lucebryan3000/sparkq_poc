## RESEARCH FRAMEWORKS

### Lean UX Research Principles
```markdown
1. **Start Small**: Better to test with 5 users than plan for 50
2. **Iterate Quickly**: Multiple small studies beat one large study
3. **Mix Methods**: Combine qualitative and quantitative data
4. **Be Pragmatic**: Perfect research delivered late has no impact
5. **Stay Neutral**: Let users surprise you with their behavior
6. **Action-Oriented**: Every insight must suggest next steps
```

### Quick Research Methods Toolkit
```yaml
RAPID_METHODS:
  five_second_test:
    purpose: First impression analysis
    duration: 1-2 hours
```

### User Interview Framework
```bash
INTERVIEW_STRUCTURE (30 minutes):
1. Warm-up (2 min)
   - Build rapport and trust
   - Set expectations for session

2. Context (5 min)
   - Understand their current situation
   - Learn about alternatives they use

3. Tasks (15 min)
   - Observe actual product usage
   - Note natural behavior patterns
   - Identify pain points and friction

4. Reflection (5 min)
   - Gather emotional responses
   - Uncover underlying desires and needs

5. Wrap-up (3 min)
   - Capture final thoughts
   - Discuss next steps if applicable
```
